    🔀 World 5a — 조건문 갈림길 해설¶

    if, elif, else를 사용하여 조건에 따라 다른 동작을 수행합니다. "프로그램이 스스로 판단한다"는 경험을 통해 컴퓨팅 사고의 핵심인 조건적 사고를 기릅니다.

    🧩 성인 확인 (r13)¶

    나이를 입력받아, 18세 이상이면 "성인입니다"를 출력하세요. 미만이면 아무것도 출력하지 않습니다.

    난이도: ⭐ | 진단 초점: L1 | 전략: pattern

    풀이 1: 단순 if문 (pattern)¶

    age = int(input("나이를 입력하세요: "))
if age >= 18:
    print("성인입니다")
    • if 문의 가장 기본적인 형태: 조건이 참일 때만 실행합니다.
    • else가 없는 if문 — 조건을 만족하지 않으면 아무것도 하지 않습니다.
    • >= (크거나 같다) 비교 연산자를 사용합니다.
    • 들여쓰기(indent)가 조건문의 범위를 결정합니다.

    접근법 비교¶

    전략 코드 길이 가독성 확장성 추천 대상
    pattern (단순 if) 3줄 ★★★★★ ★★★ 모든 학생

    ⚠️ 학생들이 자주 하는 실수¶

    실수: 콜론 누락 (아키타입: syntax-stuck)

    ❌ 잘못된 코드: 

    if age >= 18
    print("성인입니다")
    ✅ 올바른 코드: 
    if age >= 18:
    print("성인입니다")
    원인: if 문 끝의 콜론(:)은 파이썬 문법의 필수 요소입니다. SyntaxError: expected ':' 오류가 발생합니다.

    실수: str과 int 비교 (아키타입: eager-beginner)

    ❌ 잘못된 코드: 

    age = input("나이를 입력하세요: ")
if age >= 18:  # TypeError! str과 int 비교
    print("성인입니다")
    ✅ 올바른 코드: 
    age = int(input("나이를 입력하세요: "))
if age >= 18:
    print("성인입니다")
    원인: input()은 문자열을 반환합니다. 문자열 "18"과 정수 18은 다릅니다.

    실수: > 대신 >= (경계값 오류) (아키타입: validation-weak)

    ❌ 잘못된 코드: 

    if age > 18:  # 18세는 성인이 아님!
    print("성인입니다")
    ✅ 올바른 코드: 
    if age >= 18:  # 18세도 성인
    print("성인입니다")
    원인: >(초과)와 >=(이상)의 차이를 무시합니다. 18세가 경계값인데, "18세 이상"이 조건이므로 >=를 사용해야 합니다.

    실수: 불필요한 else 추가 (아키타입: strategy-biased)

    ❌ 잘못된 코드: 

    if age >= 18:
    print("성인입니다")
else:
    print("미성년자입니다")  # 요구사항에 없음!
    ✅ 올바른 코드: 
    if age >= 18:
    print("성인입니다")
    원인: "if가 있으면 else도 있어야 한다"고 오해합니다. 이 문제는 else 없이 if만 사용해야 합니다.

    🎯 교사 코칭 포인트¶

    사고 수준 판단

    • L0 (실행): 18을 입력했을 때와 17을 입력했을 때의 결과 차이를 확인할 수 있는가?
    • L1 (이해): if 문의 동작 원리(조건 → 참/거짓 → 실행/건너뜀)를 설명할 수 있는가?
    • L2 (적용): "미성년자입니다"를 추가하려면 어떻게 수정해야 하는지 제안할 수 있는가?
    • L3 (전략): 경계값(17, 18, 19) 테스트의 중요성을 이해하고 직접 검증할 수 있는가?

    경계값 테스트 교수법

    이 문제에서 반드시 경계값 테스트를 경험시키세요. "17, 18, 19 세 가지 나이를 넣어볼까요? 결과가 각각 어떻게 되어야 할까요?" 이 습관은 이후 모든 조건문 문제에서 중요합니다.

    🧩 홀짝 판별기 (r14)¶

    정수를 입력받아 홀수인지 짝수인지 판별하세요.

    난이도: ⭐ | 진단 초점: L1, L2 | 전략: math, pattern

    풀이 1: 나머지 연산 (math)¶

    num = int(input("정수를 입력하세요: "))
if num % 2 == 0:
    print("짝수입니다")
else:
    print("홀수입니다")
    • % (나머지 연산자): 2로 나눈 나머지가 0이면 짝수, 1이면 홀수.
    • == (등호 비교): 값이 같은지 비교합니다. =(대입)과 구분하세요.
    • 수학적 정의를 코드로 직접 번역하는 대표적 예시입니다.

    풀이 2: 조건 표현식 (pattern)¶

    num = int(input("정수를 입력하세요: "))
result = "짝수" if num % 2 == 0 else "홀수"
print(f"{result}입니다")
    • 파이썬의 삼항 연산자(conditional expression)를 사용합니다.
    • 한 줄로 조건 판단 + 값 할당이 가능합니다.
    • 간결하지만, 초보자에게는 가독성이 떨어질 수 있습니다.

    접근법 비교¶

    전략 코드 길이 가독성 확장성 추천 대상
    math (if-else) 5줄 ★★★★★ ★★★★★ 모든 학생
    pattern (조건 표현식) 3줄 ★★★ ★★★ 심화 탐구형 학생

    ⚠️ 학생들이 자주 하는 실수¶

    실수: = 대신 == 사용 실수 (아키타입: syntax-stuck)

    ❌ 잘못된 코드: 

    if num % 2 = 0:  # SyntaxError!
    print("짝수입니다")
    ✅ 올바른 코드: 
    if num % 2 == 0:
    print("짝수입니다")
    원인: =(대입)과 ==(비교)를 혼동합니다. 조건문에서는 반드시 ==를 사용해야 합니다.

    실수: 홀수 조건을 별도 if로 작성 (아키타입: strategy-biased)

    ❌ 잘못된 코드: 

    if num % 2 == 0:
    print("짝수입니다")
if num % 2 == 1:
    print("홀수입니다")
    ✅ 올바른 코드: 
    if num % 2 == 0:
    print("짝수입니다")
else:
    print("홀수입니다")
    원인: "짝수가 아니면 홀수"라는 상호 배타적 관계를 else로 표현하지 못합니다. 음수에서 % 2 == -1이 될 수 있어 두 번째 if문이 실패할 수도 있습니다.

    🎯 교사 코칭 포인트¶

    사고 수준 판단

    • L0 (실행): 짝수와 홀수를 입력하여 결과를 확인할 수 있는가?
    • L1 (이해): % 연산자의 의미를 설명할 수 있는가? ("나누고 나머지를 구한다")
    • L2 (적용): 3의 배수 판별, 10의 배수 판별 등에 같은 패턴을 적용할 수 있는가?
    • L3 (전략): if-else vs 조건 표현식을 비교하고, 가독성 관점에서 선택 기준을 제시할 수 있는가?

    🧩 성적표 등급 (r15)¶

    점수(0~100)를 입력받아 등급(A~F)을 판정하세요. (90↑A, 80↑B, 70↑C, 60↑D, 나머지 F)

    난이도: ⭐⭐ | 진단 초점: L1, L2 | 전략: pattern

    풀이 1: if-elif-else 체인 (pattern)¶

    score = int(input("점수를 입력하세요: "))

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(f"등급: {grade}")
    • elif는 "else if"의 줄임말로, 여러 조건을 순차적으로 검사합니다.
    • 위에서 아래로 검사하므로, 큰 값부터 비교합니다.
    • score >= 80의 elif에 도달했다면, 이미 score >= 90은 거짓입니다.
    • 마지막 else는 모든 조건에 해당하지 않는 경우(60점 미만)를 처리합니다.

    접근법 비교¶

    전략 코드 길이 가독성 확장성 추천 대상
    pattern (if-elif-else) 12줄 ★★★★★ ★★★★★ 모든 학생

    왜 if 5개가 아닌 elif를 사용하는가?

    # 나쁜 예: 독립적 if 5개
if score >= 90: grade = "A"
if score >= 80: grade = "B"  # 95점도 여기 해당!
    독립적 if를 사용하면 95점인 학생이 A → B → C → D 순서로 덮어씌워져 최종적으로 "D"가 됩니다.

    ⚠️ 학생들이 자주 하는 실수¶

    실수: 조건 순서 오류 — 작은 값부터 비교 (아키타입: eager-beginner)

    ❌ 잘못된 코드: 

    if score >= 60:
    grade = "D"
elif score >= 70:
    grade = "C"  # 여기 도달 불가! 75점은 이미 위에서 D
elif score >= 80:
    grade = "B"
elif score >= 90:
    grade = "A"
else:
    grade = "F"
    ✅ 올바른 코드: 
    if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"
    원인: elif 체인에서는 큰 값부터 비교해야 합니다. 작은 값부터 비교하면 95점도 >= 60에 걸려 "D"가 됩니다.

    실수: elif 대신 if 남용 (아키타입: strategy-biased)

    ❌ 잘못된 코드: 

    if score >= 90:
    grade = "A"
if score >= 80:
    grade = "B"
if score >= 70:
    grade = "C"
    ✅ 올바른 코드: 
    if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
    원인: if를 여러 번 쓰면 각 조건이 독립적으로 실행됩니다. 95점이 A → B → C로 순차적으로 덮어씌워집니다.

    실수: 범위를 명시적으로 쓰려다 오류 (아키타입: careful-thinker)

    ❌ 잘못된 코드: 

    if score >= 90 and score <= 100:
    grade = "A"
elif score >= 80 and score <= 89:
    grade = "B"
# ... (불필요하게 복잡)
    ✅ 올바른 코드: 
    if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
    원인: elif의 "이전 조건이 거짓"이라는 암묵적 조건을 이해하지 못해, 모든 범위를 명시적으로 작성합니다. 틀린 것은 아니지만 불필요하게 복잡합니다.

    🎯 교사 코칭 포인트¶

    사고 수준 판단

    • L0 (실행): 다양한 점수(95, 85, 75, 55)를 넣어 등급을 확인할 수 있는가?
    • L1 (이해): elif가 "이전 조건이 거짓일 때만 검사한다"는 것을 이해하는가?
    • L2 (적용): 등급 기준을 변경하거나, A+/A0/A- 등 세분화할 수 있는가?
    • L3 (전략): elif 순서가 왜 중요한지, if 5개와 elif 체인의 차이를 논리적으로 설명할 수 있는가?

    "만약 점수가 100점 초과거나 0점 미만이면 어떻게 처리할까요?" 입력 검증(validation)의 필요성을 자연스럽게 도입할 수 있습니다.
