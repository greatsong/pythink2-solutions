# ⚓ World 3 — 입력 항구 해설

> 사용자로부터 값을 입력받아 처리하는 법을 배웁니다.
> "프로그램이 외부와 소통한다"는 개념, 그리고 `input()`의 반환 타입(str)을 이해하는 것이 핵심입니다.

---

## 🧩 인사하는 프로그램 (r07)

> `input()`으로 이름을 입력받고, f-string으로 인사말을 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1 | **전략**: pattern

### 풀이 1: input + f-string (pattern)

=== "코드"
    ```python
    name = input("이름을 입력하세요: ")
    print(f"안녕하세요, {name}님! 반갑습니다.")
    ```

=== "핵심 아이디어"
    - `input()` 함수는 사용자 입력을 **문자열(str)**로 반환합니다.
    - 입력받은 값을 변수에 저장하고, f-string으로 출력에 활용합니다.
    - `input()` 안의 문자열은 프롬프트(안내 메시지)입니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (input + f-string) | 2줄 | ★★★★★ | ★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: input() 결과를 변수에 저장하지 않음 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    input("이름을 입력하세요: ")
    print(f"안녕하세요, {name}님!")  # NameError!
    ```
    **✅ 올바른 코드:**
    ```python
    name = input("이름을 입력하세요: ")
    print(f"안녕하세요, {name}님!")
    ```
    **원인**: `input()`의 반환값을 변수에 저장하지 않으면, 입력받은 값을 사용할 수 없습니다.

!!! danger "실수: 프롬프트 없는 input (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    name = input()
    ```
    **✅ 올바른 코드:**
    ```python
    name = input("이름을 입력하세요: ")
    ```
    **원인**: 프로그램이 멈춘 것처럼 보여 사용자가 당황합니다. 기능상 문제는 없지만, 사용자 경험(UX) 관점에서 안내 메시지가 필요합니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 프로그램을 실행하고, 입력창에 이름을 입력할 수 있는가?
    - **L1 (이해)**: `input()`이 값을 "받아온다"는 개념을 이해하는가?
    - **L2 (적용)**: 이름 외에 다른 정보(학교, 취미 등)도 입력받도록 확장할 수 있는가?
    - **L3 (전략)**: 이 단계에서는 해당하지 않음

## 🧩 태어난 해 맞추기 (r08)

> 나이를 입력받아 태어난 해(2026 - 나이)를 계산하여 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1, L2 | **전략**: math

### 풀이 1: int 변환 + 뺄셈 (math)

=== "코드"
    ```python
    age = int(input("나이를 입력하세요: "))
    birth_year = 2026 - age
    print(f"태어난 해: {birth_year}년")
    ```

=== "핵심 아이디어"
    - `input()`은 항상 **문자열**을 반환합니다.
    - 숫자 계산을 하려면 `int()`로 정수 변환이 필요합니다.
    - `int(input(...))`은 "입력받아서 바로 정수로 변환"하는 관용구입니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| math (int 변환 + 계산) | 3줄 | ★★★★★ | ★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 괄호 안 닫힘 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    age = int(input("나이를 입력하세요: ")
    ```
    **✅ 올바른 코드:**
    ```python
    age = int(input("나이를 입력하세요: "))
    ```
    **원인**: `int()`와 `input()` 괄호가 중첩되어 닫는 괄호를 하나 빠뜨립니다. `int(input(...))` 패턴에서 닫는 괄호는 **두 개**여야 합니다.

!!! danger "실수: str에 뺄셈 시도 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    age = input("나이를 입력하세요: ")
    birth_year = 2026 - age  # TypeError!
    ```
    **✅ 올바른 코드:**
    ```python
    age = int(input("나이를 입력하세요: "))
    birth_year = 2026 - age
    ```
    **원인**: `input()`의 반환값은 문자열입니다. 문자열에 뺄셈을 하면 `TypeError: unsupported operand type(s) for -: 'int' and 'str'` 오류가 발생합니다.

!!! danger "실수: 뺄셈 방향 반대 (아키타입: struggling-failure)"
    **❌ 잘못된 코드:**
    ```python
    age = int(input("나이를 입력하세요: "))
    birth_year = age - 2026  # 음수가 나옴!
    ```
    **✅ 올바른 코드:**
    ```python
    age = int(input("나이를 입력하세요: "))
    birth_year = 2026 - age
    ```
    **원인**: "올해 연도에서 나이를 빼야" 태어난 해가 나옵니다. 반대로 빼면 음수가 됩니다.

!!! danger "실수: 연도 오류 (아024 또는 2025 사용) (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    birth_year = 2025 - age  # 작년 연도!
    ```
    **✅ 올바른 코드:**
    ```python
    birth_year = 2026 - age
    ```
    **원인**: 현재 연도(2026)를 잘못 입력합니다. 작년 연도를 사용하거나, 연도를 직접 치다 오타가 나는 경우입니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 나이를 입력하여 결과를 확인할 수 있는가?
    - **L1 (이해)**: `int()`가 왜 필요한지, `input()`이 문자열을 반환하는 이유를 설명할 수 있는가?
    - **L2 (적용)**: 생년 대신 졸업 연도, 100세까지 남은 해 등 유사한 계산에 적용할 수 있는가?
    - **L3 (전략)**: "만 나이"와 "세는 나이"의 차이를 고려한 개선점을 제안할 수 있는가?

!!! tip "TypeError 교수 전략"
    이 문제는 `TypeError`를 **의도적으로 경험**시킬 수 있는 좋은 기회입니다.
    먼저 `int()` 없이 실행해보게 하고, 오류 메시지를 함께 읽으며 "왜 타입 변환이 필요한가"를 체감하도록 유도하세요.

## 🧩 피자 주문서 (r09)

> 피자 이름(str), 수량(int), 가격(int)을 각각 입력받아 주문서를 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1, L2 | **전략**: decomposition

### 풀이 1: 단계별 입력과 계산 (decomposition)

=== "코드"
    ```python
    # 1단계: 입력
    pizza = input("피자 이름: ")
    count = int(input("수량: "))
    price = int(input("가격: "))

    # 2단계: 계산
    total = count * price

    # 3단계: 출력
    print("=== 주문서 ===")
    print(f"피자: {pizza}")
    print(f"수량: {count}판")
    print(f"단가: {price}원")
    print(f"합계: {total}원")
    ```

=== "핵심 아이디어"
    - 여러 개의 `input()`을 사용하여 다양한 타입의 데이터를 입력받습니다.
    - 문자열은 그대로, 숫자는 `int()`로 변환합니다.
    - **입력 → 계산 → 출력** 3단계 구조로 문제를 분해합니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| decomposition (3단계 분해) | 10줄 | ★★★★★ | ★★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 숫자 입력에 int() 누락 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    count = input("수량: ")
    price = input("가격: ")
    total = count * price  # str * str → TypeError!
    ```
    **✅ 올바른 코드:**
    ```python
    count = int(input("수량: "))
    price = int(input("가격: "))
    total = count * price
    ```
    **원인**: r08에서 배운 `int()` 변환을 여러 변수에 일관되게 적용하지 못합니다.

!!! danger "실수: 문자열 입력에 int() 적용 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    pizza = int(input("피자 이름: "))  # ValueError!
    ```
    **✅ 올바른 코드:**
    ```python
    pizza = input("피자 이름: ")
    ```
    **원인**: r08에서 `int(input(...))`을 배운 뒤, 모든 `input()`에 `int()`를 붙여야 한다고 오해합니다.

!!! danger "실수: 곱셈 대신 덧셈 (아키타입: struggling-failure)"
    **❌ 잘못된 코드:**
    ```python
    total = count + price  # 수량 + 가격 = 의미 없는 값
    ```
    **✅ 올바른 코드:**
    ```python
    total = count * price
    ```
    **원인**: "합계"라는 단어에서 "더하기"를 연상합니다. 문제의 의미(단가 x 수량)를 정확히 파악하지 못한 경우입니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 세 가지 입력을 순서대로 넣고 결과를 확인할 수 있는가?
    - **L1 (이해)**: 어떤 입력에 `int()`가 필요하고, 어떤 입력에는 필요 없는지 구분할 수 있는가?
    - **L2 (적용)**: 배달비, 할인율 등 추가 항목을 넣어 주문서를 확장할 수 있는가?
    - **L3 (전략)**: 입력-계산-출력 3단계 패턴을 다른 문제에도 적용할 수 있는가?

!!! tip "decomposition 사고력 강화"
    이 문제는 **분해적 사고(decomposition)**를 처음 경험하는 좋은 기회입니다.
    "이 문제를 세 덩어리로 나눌 수 있을까요?" 라는 질문으로 학생이 스스로 구조를 발견하도록 유도하세요.
