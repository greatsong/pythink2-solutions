# ⚙️ World 4 — 연산 공장 해설

> 사칙연산, 정수 나눗셈(`//`), 나머지(`%`) 등 파이썬의 연산자를 본격적으로 활용합니다.
> "계산을 코드로 표현한다"는 경험을 통해 수학과 프로그래밍의 연결고리를 만드는 월드입니다.

## 🧩 사칙연산 계산기 (r10)

> 두 수를 입력받아 덧셈, 뺄셈, 곱셈, 정수 나눗셈 결과를 모두 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1 | **전략**: pattern

### 풀이 1: 네 가지 연산 출력 (pattern)

=== "코드"
    ```python
    a = int(input("첫 번째 수: "))
    b = int(input("두 번째 수: "))

    print(f"{a} + {b} = {a + b}")
    print(f"{a} - {b} = {a - b}")
    print(f"{a} * {b} = {a * b}")
    print(f"{a} // {b} = {a // b}")
    ```

=== "핵심 아이디어"
    - 네 가지 연산(`+`, `-`, `*`, `//`)을 각각 f-string 안에서 바로 계산합니다.
    - `//`는 정수 나눗셈(몫)으로, `/`(실수 나눗셈)와 구분됩니다.
    - f-string 중괄호 안에서 직접 연산이 가능합니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (f-string 내 직접 연산) | 6줄 | ★★★★★ | ★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: / 대신 // 미사용 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    print(f"{a} / {b} = {a / b}")  # 출력: 10 / 3 = 3.3333...
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"{a} // {b} = {a // b}")  # 출력: 10 // 3 = 3
    ```
    **원인**: `/`와 `//`의 차이를 모릅니다. `/`는 실수 나눗셈(float), `//`는 정수 나눗셈(몫)입니다.

!!! danger "실수: int() 변환 누락 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    a = input("첫 번째 수: ")
    b = input("두 번째 수: ")
    print(f"{a} + {b} = {a + b}")  # 문자열 연결: "3" + "5" = "35"
    ```
    **✅ 올바른 코드:**
    ```python
    a = int(input("첫 번째 수: "))
    b = int(input("두 번째 수: "))
    print(f"{a} + {b} = {a + b}")  # 숫자 덧셈: 3 + 5 = 8
    ```
    **원인**: 문자열 `+`는 **연결(concatenation)**입니다. "3" + "5" = "35"가 되어 의도와 다른 결과가 나옵니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 두 수를 입력하고 네 가지 결과를 확인할 수 있는가?
    - **L1 (이해)**: `/`와 `//`의 차이, 문자열 `+`와 숫자 `+`의 차이를 설명할 수 있는가?
    - **L2 (적용)**: 나머지(`%`), 거듭제곱(`**`) 등을 추가하여 확장할 수 있는가?
    - **L3 (전략)**: 이 단계에서는 해당하지 않음

## 🧩 시간 변환기 (r11)

> 분(minutes)을 입력받아 시간과 분으로 변환하여 출력하세요. (예: 130분 → 2시간 10분)
>
> **난이도**: ⭐ | **진단 초점**: L1, L2 | **전략**: math

### 풀이 1: 몫과 나머지 활용 (math)

=== "코드"
    ```python
    minutes = int(input("분을 입력하세요: "))
    hours = minutes // 60
    mins = minutes % 60
    print(f"{minutes}분은 {hours}시간 {mins}분입니다.")
    ```

=== "핵심 아이디어"
    - `//` (정수 나눗셈): 60으로 나눈 **몫** = 시간
    - `%` (나머지 연산): 60으로 나눈 **나머지** = 남은 분
    - `//`와 `%`는 "단위 변환"의 핵심 도구입니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| math (몫/나머지) | 4줄 | ★★★★★ | ★★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: f-string 안 중괄호 혼동 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    print(f"{minutes}분은 {hours}시간 {mins}분입니다.")
    # 이 코드 자체는 맞지만, 변수명 오타가 많음
    print(f"{minute}분은 {hour}시간 {min}분입니다.")  # NameError!
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"{minutes}분은 {hours}시간 {mins}분입니다.")
    ```
    **원인**: 변수명의 단수/복수를 혼동하거나, `min`(내장 함수)과 `mins`(사용자 변수)를 혼동합니다.

!!! danger "실수: input()에 int() 누락 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    minutes = input("분을 입력하세요: ")
    hours = minutes // 60  # TypeError!
    ```
    **✅ 올바른 코드:**
    ```python
    minutes = int(input("분을 입력하세요: "))
    hours = minutes // 60
    ```
    **원인**: 문자열에 `//` 연산을 시도합니다. `TypeError`가 발생합니다.

!!! danger "실수: //와 % 반대로 사용 (아키타입: struggling-failure)"
    **❌ 잘못된 코드:**
    ```python
    hours = minutes % 60   # 나머지가 시간에?
    mins = minutes // 60   # 몫이 분에?
    ```
    **✅ 올바른 코드:**
    ```python
    hours = minutes // 60  # 몫 = 시간
    mins = minutes % 60    # 나머지 = 분
    ```
    **원인**: `//`(몫)와 `%`(나머지)의 의미를 반대로 기억합니다. 130분의 경우 `130 % 60 = 10`이 시간으로 출력되어 "10시간 2분"이 됩니다.

!!! danger "실수: 출력 형식에 공백 누락 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    print(f"{minutes}분은{hours}시간{mins}분입니다.")
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"{minutes}분은 {hours}시간 {mins}분입니다.")
    ```
    **원인**: 공백 하나 차이지만, 출력 형식을 정확히 맞추는 습관이 중요합니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 130분 입력 시 2시간 10분이 나오는 것을 확인할 수 있는가?
    - **L1 (이해)**: `//`와 `%`의 의미를 일상 언어로 설명할 수 있는가? ("60으로 나눠서 몫은 시간, 나머지는 분")
    - **L2 (적용)**: 초→분·초 변환, 원→만원 단위 변환 등에 같은 패턴을 적용할 수 있는가?
    - **L3 (전략)**: `divmod()` 함수를 활용한 대안을 탐색할 수 있는가?

!!! tip "시각적 비유"
    "130개의 사탕을 60개씩 봉지에 담으면, 봉지 2개(몫=시간)와 남은 10개(나머지=분)가 돼요"
    이런 비유로 `//`와 `%`의 관계를 시각화하면 효과적입니다.

## 🧩 BMI 계산기 (r12)

> 키(cm)와 몸무게(kg)를 입력받아 BMI를 계산하세요. (BMI = 몸무게 / 키(m)^2, 소수점 1자리)
>
> **난이도**: ⭐⭐ | **진단 초점**: L1, L2 | **전략**: math, decomposition

### 풀이 1: 한 번에 계산 (math)

=== "코드"
    ```python
    height = int(input("키(cm): "))
    weight = int(input("몸무게(kg): "))

    bmi = round(weight / (height / 100) ** 2, 1)
    print(f"BMI: {bmi}")
    ```

=== "핵심 아이디어"
    - cm를 m로 변환: `height / 100`
    - BMI 공식: `weight / height_m ** 2`
    - `round(값, 1)`으로 소수점 첫째 자리까지 반올림합니다.
    - 괄호의 위치가 중요합니다: `(height / 100) ** 2`

### 풀이 2: 단계별 분해 (decomposition)

=== "코드"
    ```python
    height = int(input("키(cm): "))
    weight = int(input("몸무게(kg): "))

    # 1단계: cm → m 변환
    height_m = height / 100

    # 2단계: BMI 계산
    bmi = weight / (height_m ** 2)

    # 3단계: 반올림
    bmi = round(bmi, 1)

    # 4단계: 출력
    print(f"BMI: {bmi}")
    ```

=== "비교"
    - 각 단계가 독립적으로 확인 가능합니다.
    - 중간 변수(`height_m`)로 단위 변환 과정을 명시합니다.
    - 디버깅이 쉽습니다 (어느 단계에서 문제가 생겼는지 파악 가능).
    - 계산 순서를 실수할 위험이 줄어듭니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| math (한 줄 계산) | 4줄 | ★★★ | ★★★ | 수학 공식에 익숙한 학생 |
| decomposition (단계별) | 10줄 | ★★★★★ | ★★★★★ | 초보~중급 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 괄호 위치 오류 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    bmi = weight / height / 100 ** 2  # 100만 제곱됨!
    ```
    **✅ 올바른 코드:**
    ```python
    bmi = weight / (height / 100) ** 2
    ```
    **원인**: 연산자 우선순위에 의해 `100 ** 2 = 10000`이 먼저 계산되고, `weight / height / 10000`이 됩니다. 의도와 전혀 다른 값이 나옵니다.

!!! danger "실수: cm를 m로 변환하지 않음 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    bmi = round(weight / height ** 2, 1)  # 키가 cm 단위인 채로!
    ```
    **✅ 올바른 코드:**
    ```python
    bmi = round(weight / (height / 100) ** 2, 1)
    ```
    **원인**: BMI 공식에서 키는 미터(m) 단위여야 합니다. 170cm를 그대로 넣으면 BMI가 0.00...이 됩니다.

!!! danger "실수: round() 미사용 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    bmi = weight / (height / 100) ** 2
    print(f"BMI: {bmi}")  # 출력: BMI: 24.221453287197235
    ```
    **✅ 올바른 코드:**
    ```python
    bmi = round(weight / (height / 100) ** 2, 1)
    print(f"BMI: {bmi}")  # 출력: BMI: 24.2
    ```
    **원인**: 소수점 1자리 요구사항을 놓칩니다. 결과 확인 없이 제출하는 패턴입니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 키와 몸무게를 입력하여 BMI 값을 확인할 수 있는가?
    - **L1 (이해)**: 단위 변환(cm→m)이 왜 필요한지, `round()` 함수의 역할을 설명할 수 있는가?
    - **L2 (적용)**: BMI 결과에 따른 판정(저체중/정상/과체중)을 추가할 수 있는가? (→ World 5a 연결)
    - **L3 (전략)**: 한 줄 계산과 단계별 분해의 장단점을 비교하고, 팀 프로젝트에서 어떤 방식이 적합한지 판단할 수 있는가?

!!! tip "수학 교과 연결"
    BMI 공식은 물리(단위 변환), 수학(비율, 제곱) 개념이 통합됩니다.
    "키를 미터로 바꾸지 않으면 어떤 일이 일어날까요?" 라는 질문으로 단위의 중요성을 체감하게 하세요.
