# 🎠 World 6 — 반복문 놀이터 해설

> `for`와 `while` 반복문으로 같은 작업을 여러 번 수행합니다.
> "반복을 코드 한 줄로 표현한다"는 프로그래밍의 강력함을 체감하는 월드입니다.

---

## 🧩 구구단 한 단 (r19)

> 정수 n을 입력받아 n단 구구단(1~9)을 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1, L2 | **전략**: pattern

### 풀이 1: for 반복문 (pattern)

=== "코드"
    ```python
    n = int(input("몇 단? "))
    for i in range(1, 10):
        print(f"{n} x {i} = {n * i}")
    ```

=== "핵심 아이디어"
    - `range(1, 10)`: 1부터 9까지의 숫자를 생성합니다 (10은 미포함).
    - `for i in range(...)`: 변수 `i`가 1, 2, 3, ..., 9로 순차적으로 변합니다.
    - f-string 안에서 `n * i` 계산을 직접 수행합니다.
    - 구구단은 for문의 가장 대표적인 예시입니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (for + range) | 3줄 | ★★★★★ | ★★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: for 문 끝에 콜론 누락 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    for i in range(1, 10)
        print(f"{n} x {i} = {n * i}")
    ```
    **✅ 올바른 코드:**
    ```python
    for i in range(1, 10):
        print(f"{n} x {i} = {n * i}")
    ```
    **원인**: `if`문과 마찬가지로, `for`문 끝에도 콜론(`:`)이 필요합니다. `SyntaxError: expected ':'` 오류가 발생합니다.

!!! danger "실수: str과 int 곱셈 혼동 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    n = input("몇 단? ")  # str
    for i in range(1, 10):
        print(f"{n} x {i} = {n * i}")  # "3" * 2 = "33" (문자열 반복!)
    ```
    **✅ 올바른 코드:**
    ```python
    n = int(input("몇 단? "))
    for i in range(1, 10):
        print(f"{n} x {i} = {n * i}")  # 3 * 2 = 6
    ```
    **원인**: 문자열 `"3" * 2`는 `"33"`이 됩니다 (문자열 반복). `int()` 변환이 누락되면 구구단 결과가 "333333333" 같은 문자열이 됩니다.

!!! danger "실수: range(1, 9) — 9단까지 출력 안 됨 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    for i in range(1, 9):  # 1~8만 출력!
        print(f"{n} x {i} = {n * i}")
    ```
    **✅ 올바른 코드:**
    ```python
    for i in range(1, 10):  # 1~9 출력
        print(f"{n} x {i} = {n * i}")
    ```
    **원인**: `range(1, 9)`는 1~8까지만 생성합니다. `range(시작, 끝)`에서 **끝 값은 미포함**이라는 규칙을 기억해야 합니다.

!!! danger "실수: range(0, 9) — 0부터 시작 (아키타입: struggling-failure)"
    **❌ 잘못된 코드:**
    ```python
    for i in range(0, 9):  # 0부터 시작! (n x 0 = 0)
        print(f"{n} x {i} = {n * i}")
    ```
    **✅ 올바른 코드:**
    ```python
    for i in range(1, 10):  # 1부터 시작
        print(f"{n} x {i} = {n * i}")
    ```
    **원인**: `range()`의 기본 시작값이 0이라는 것을 알고 있지만, 구구단은 1부터 시작해야 합니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 3을 입력하여 3단이 출력되는 것을 확인할 수 있는가?
    - **L1 (이해)**: `range(1, 10)`이 왜 1부터 9까지인지 설명할 수 있는가?
    - **L2 (적용)**: 2~9단 전체를 출력하도록 확장할 수 있는가? (이중 for문)
    - **L3 (전략)**: 구구단 외에 "1+2+...+n" 같은 누적 계산에 for문을 적용할 수 있는가?

!!! tip "range의 '울타리 기둥' 비유"
    `range(1, 10)`의 끝값 미포함 규칙은 **울타리 기둥** 비유로 설명하세요.
    "1번 기둥에서 출발해서 10번 기둥 **직전**에 멈춰요. 그래서 9번 기둥까지만 가는 거예요."

## 🧩 카운트다운 타이머 (r20)

> 양의 정수 n을 입력받아 n부터 1까지 카운트다운 후 "발사!"를 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1, L2 | **전략**: pattern, brute_force

### 풀이 1: while 반복문 (pattern)

=== "코드"
    ```python
    n = int(input("시작 숫자: "))
    while n > 0:
        print(n)
        n -= 1
    print("발사!")
    ```

=== "핵심 아이디어"
    - `while` 문: 조건이 참인 동안 계속 반복합니다.
    - `n -= 1`: n을 1씩 줄입니다 (`n = n - 1`과 같음).
    - 반복이 끝난 후(n이 0이 되면) "발사!"를 출력합니다.
    - `print("발사!")`는 while 블록 **바깥**에 있어야 합니다 (들여쓰기 주의).

### 풀이 2: for + range 역순 (pattern)

=== "코드"
    ```python
    n = int(input("시작 숫자: "))
    for i in range(n, 0, -1):
        print(i)
    print("발사!")
    ```

=== "비교"
    - `range(n, 0, -1)`: n부터 1까지 1씩 감소하는 수열.
    - while보다 범위가 명확하고, 무한 루프 위험이 없습니다.
    - 원본 변수 `n`이 변경되지 않습니다.

### 풀이 3: 하나씩 나열 — 작은 수만 가능 (brute_force)

=== "코드"
    ```python
    n = int(input("시작 숫자: "))
    # n이 5라고 가정하면...
    print(5)
    print(4)
    print(3)
    print(2)
    print(1)
    print("발사!")
    ```

=== "비교"
    - 반복문 없이 하나씩 나열하는 방식입니다.
    - n이 고정된 경우에만 가능하며, n이 바뀌면 코드도 바꿔야 합니다.
    - **왜 반복문이 필요한지** 체감시키기 위한 비교용 코드입니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (while) | 5줄 | ★★★★ | ★★★★★ | while문 학습 중인 학생 |
| pattern (for + range) | 4줄 | ★★★★★ | ★★★★★ | for문에 익숙한 학생 |
| brute_force (나열) | n+1줄 | ★★★ | ★ | 반복문 도입 전 비교용 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: n -= 1 누락으로 무한 루프 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    n = int(input("시작 숫자: "))
    while n > 0:
        print(n)
        # n -= 1 을 빠뜨림! → 무한 루프
    print("발사!")
    ```
    **✅ 올바른 코드:**
    ```python
    n = int(input("시작 숫자: "))
    while n > 0:
        print(n)
        n -= 1
    print("발사!")
    ```
    **원인**: while문에서 조건 변수를 변경하지 않으면, 조건이 영원히 참이 되어 프로그램이 멈추지 않습니다.

!!! danger "실수: '발사!'가 반복문 안에 있음 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    while n > 0:
        print(n)
        n -= 1
        print("발사!")  # 매번 출력됨!
    ```
    **✅ 올바른 코드:**
    ```python
    while n > 0:
        print(n)
        n -= 1
    print("발사!")  # 반복 끝난 후 한 번만
    ```
    **원인**: 들여쓰기가 while 블록의 범위를 결정합니다. "발사!"가 들여쓰기 안에 있으면 매 반복마다 출력됩니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 5를 입력하여 5, 4, 3, 2, 1, 발사!가 나오는 것을 확인할 수 있는가?
    - **L1 (이해)**: while문의 조건 검사 → 실행 → 조건 검사 사이클을 설명할 수 있는가?
    - **L2 (적용)**: 2씩 감소하는 카운트다운, 짝수만 출력 등으로 변형할 수 있는가?
    - **L3 (전략)**: while vs for 각각의 장단점을 비교할 수 있는가? ("언제 while을 쓰고, 언제 for를 쓸까요?")

!!! tip "무한 루프 교수 전략"
    무한 루프는 **의도적으로 경험**시키면 효과적입니다.
    `n -= 1`을 빼고 실행해보게 한 뒤, "왜 멈추지 않을까요?"를 함께 분석하세요.
    pythink2 환경에서는 실행 시간 제한이 있어 자동으로 중단됩니다.

## 🧩 별 직사각형 (r21)

> 행(rows)과 열(cols) 수를 입력받아 별(`*`)로 직사각형을 그리세요.
>
> **난이도**: ⭐⭐ | **진단 초점**: L2, L3 | **전략**: pattern, brute_force

### 풀이 1: 문자열 곱셈 (pattern)

=== "코드"
    ```python
    rows = int(input("행: "))
    cols = int(input("열: "))
    for i in range(rows):
        print("*" * cols)
    ```

=== "핵심 아이디어"
    - `"*" * cols`: 문자열 곱셈으로 `*`을 cols번 반복합니다.
    - `for i in range(rows)`: rows번 반복하여 각 행을 출력합니다.
    - 파이썬의 문자열 곱셈을 활용한 간결한 풀이입니다.

### 풀이 2: 이중 for 반복문 (brute_force)

=== "코드"
    ```python
    rows = int(input("행: "))
    cols = int(input("열: "))
    for i in range(rows):
        line = ""
        for j in range(cols):
            line += "*"
        print(line)
    ```

=== "비교"
    - 바깥 for문: 행(row) 반복
    - 안쪽 for문: 열(column) 반복 — 한 글자씩 추가
    - 문자열 곱셈 없이도 동일한 결과를 만들 수 있습니다.
    - **이중 반복문**의 구조를 명시적으로 보여줍니다.
    - 삼각형, 다이아몬드 등 복잡한 패턴으로 확장하기 쉽습니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (문자열 곱셈) | 4줄 | ★★★★★ | ★★★ | 파이썬 문법 활용형 학생 |
| brute_force (이중 for) | 6줄 | ★★★★ | ★★★★★ | 알고리즘적 사고 연습 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: rows와 cols 반대로 사용 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    for i in range(cols):      # 열 수만큼 반복 (행이어야 함)
        print("*" * rows)      # 행 수만큼 별 (열이어야 함)
    ```
    **✅ 올바른 코드:**
    ```python
    for i in range(rows):      # 행 수만큼 반복
        print("*" * cols)      # 열 수만큼 별
    ```
    **원인**: "행"과 "열"의 의미를 혼동합니다. 행(row)은 세로 방향 개수, 열(col)은 가로 방향 개수입니다.

!!! danger "실수: print가 for문 바깥 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    for i in range(rows):
        line = "*" * cols
    print(line)  # 마지막 줄만 한 번 출력!
    ```
    **✅ 올바른 코드:**
    ```python
    for i in range(rows):
        print("*" * cols)
    ```
    **원인**: `print`가 for문 블록 바깥에 있으면, 반복이 모두 끝난 후 마지막 값만 한 번 출력됩니다.

!!! danger "실수: 이중 for에서 줄바꿈 처리 미흡 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    for i in range(rows):
        for j in range(cols):
            print("*")  # 매 글자마다 줄바꿈!
    ```
    **✅ 올바른 코드:**
    ```python
    for i in range(rows):
        for j in range(cols):
            print("*", end="")  # 줄바꿈 없이
        print()  # 한 행 끝나면 줄바꿈
    ```
    **원인**: `print()`는 기본적으로 줄바꿈을 추가합니다. `end=""`를 사용하여 줄바꿈을 억제해야 합니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: rows=3, cols=5를 입력하여 3x5 직사각형이 나오는 것을 확인할 수 있는가?
    - **L1 (이해)**: for문이 "몇 번 반복하는지"를 설명할 수 있는가?
    - **L2 (적용)**: 별 대신 다른 문자, 또는 직각삼각형 패턴을 만들 수 있는가?
    - **L3 (전략)**: 문자열 곱셈과 이중 for문을 비교하고, 각각 어떤 패턴에 적합한지 판단할 수 있는가?

!!! tip "패턴 확장 질문"
    직사각형을 성공한 학생에게는 다음 질문을 던져보세요:
    - "직각삼각형은 어떻게 만들까요?" (`"*" * (i+1)`)
    - "역삼각형은 어떻게 할까요?" (`"*" * (rows-i)`)
    이런 확장이 **패턴 인식** 능력을 키워줍니다.
