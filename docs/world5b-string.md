# 🌲 World 5b — 문자열 숲 해설

> 문자열의 인덱싱, 슬라이싱, 메서드를 활용하여 텍스트를 자유자재로 다룹니다.
> "데이터를 원하는 형태로 변환한다"는 프로그래밍의 핵심 능력을 기르는 월드입니다.

---

## 🧩 글자 뽑기 기계 (r16)

> 문자열에서 첫 글자, 마지막 글자, 전체 길이를 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1 | **전략**: pattern

### 풀이 1: 인덱싱과 len() (pattern)

=== "코드"
    ```python
    text = input("문자열을 입력하세요: ")
    print(f"첫 글자: {text[0]}")
    print(f"마지막 글자: {text[-1]}")
    print(f"글자 수: {len(text)}")
    ```

=== "핵심 아이디어"
    - `text[0]`: 첫 번째 글자 (인덱스는 0부터 시작).
    - `text[-1]`: 마지막 글자 (음수 인덱스는 뒤에서부터).
    - `len(text)`: 문자열의 길이(글자 수)를 반환합니다.
    - 인덱싱은 대괄호 `[]`를 사용합니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (인덱싱 + len) | 4줄 | ★★★★★ | ★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 인덱스 1부터 시작으로 오해 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    print(f"첫 글자: {text[1]}")  # 두 번째 글자!
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"첫 글자: {text[0]}")
    ```
    **원인**: 일상생활에서 "첫 번째 = 1"이지만, 프로그래밍에서는 "첫 번째 = 0"입니다. 이 **제로 인덱싱(zero-indexing)** 개념이 처음에는 혼란을 줍니다.

!!! danger "실수: 마지막 글자를 len으로 접근 (아키타입: strategy-biased)"
    **❌ 잘못된 코드:**
    ```python
    print(f"마지막 글자: {text[len(text)]}")  # IndexError!
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"마지막 글자: {text[-1]}")
    # 또는
    print(f"마지막 글자: {text[len(text)-1]}")
    ```
    **원인**: 길이가 5인 문자열의 인덱스는 0~4입니다. `text[5]`는 범위를 벗어납니다. 음수 인덱스 `-1`이 훨씬 간결합니다.

!!! danger "실수: 괄호와 대괄호 혼동 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    print(f"첫 글자: {text(0)}")  # TypeError: 'str' is not callable
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"첫 글자: {text[0]}")
    ```
    **원인**: `()`는 함수 호출, `[]`는 인덱싱입니다. 문자열에 `()`를 쓰면 "함수처럼 호출"하려다 오류가 납니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: "Hello"를 입력하여 H, o, 5가 출력되는 것을 확인할 수 있는가?
    - **L1 (이해)**: 인덱스 0이 왜 첫 글자인지, `-1`이 왜 마지막인지 설명할 수 있는가?
    - **L2 (적용)**: 두 번째 글자, 뒤에서 두 번째 글자도 뽑을 수 있는가?
    - **L3 (전략)**: 이 단계에서는 해당하지 않음

!!! tip "시각적 교수법"
    문자열 인덱싱은 **번호판**에 비유하면 효과적입니다.
    ```
    문자열:  H   e   l   l   o
    양수:    0   1   2   3   4
    음수:   -5  -4  -3  -2  -1
    ```
    이 표를 화이트보드에 그리며 설명하면 직관적으로 이해합니다.

## 🧩 비밀 메시지 변환기 (r17)

> 문자열을 대문자, 소문자로 변환하고, 공백을 `*`로 바꿔 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1 | **전략**: pattern

### 풀이 1: 문자열 메서드 활용 (pattern)

=== "코드"
    ```python
    text = input("메시지를 입력하세요: ")
    print(f"대문자: {text.upper()}")
    print(f"소문자: {text.lower()}")
    print(f"비밀코드: {text.replace(' ', '*')}")
    ```

=== "핵심 아이디어"
    - `.upper()`: 모든 글자를 대문자로 변환한 **새 문자열** 반환.
    - `.lower()`: 모든 글자를 소문자로 변환한 새 문자열 반환.
    - `.replace(old, new)`: 특정 문자를 다른 문자로 바꾼 새 문자열 반환.
    - 원본 문자열(`text`)은 변경되지 않습니다 (문자열은 **불변(immutable)**).

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (메서드 체인) | 4줄 | ★★★★★ | ★★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 메서드 호출 시 괄호 누락 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    print(f"대문자: {text.upper}")  # 메서드 객체가 출력됨
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"대문자: {text.upper()}")
    ```
    **원인**: `text.upper`는 메서드 자체를 참조하는 것이고, `text.upper()`는 메서드를 **호출**하는 것입니다. 괄호가 없으면 `<built-in method upper ...>` 같은 문자열이 출력됩니다.

!!! danger "실수: replace 인자 순서 반대 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    print(f"비밀코드: {text.replace('*', ' ')}")  # *을 공백으로! (반대)
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"비밀코드: {text.replace(' ', '*')}")   # 공백을 *로
    ```
    **원인**: `replace(old, new)`에서 첫 번째가 "바꿀 대상", 두 번째가 "바꿀 값"입니다.

!!! danger "실수: 원본이 변경된다고 오해 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    text.upper()
    print(text)  # 여전히 소문자!
    ```
    **✅ 올바른 코드:**
    ```python
    upper_text = text.upper()
    print(upper_text)
    # 또는 바로 출력
    print(text.upper())
    ```
    **원인**: 문자열 메서드는 **새 문자열을 반환**하며, 원본을 변경하지 않습니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: "Hello World"를 입력하여 세 가지 변환 결과를 확인할 수 있는가?
    - **L1 (이해)**: `.upper()`, `.lower()`, `.replace()`의 역할을 각각 설명할 수 있는가?
    - **L2 (적용)**: 다른 문자 치환(예: 모음 제거, 특정 단어 교체)에 `.replace()`를 활용할 수 있는가?
    - **L3 (전략)**: 이 단계에서는 해당하지 않음

## 🧩 단어 자르기 도구 (r18)

> 문자열의 앞 3글자, 뒤 3글자, 뒤집은 문자열을 출력하세요.
>
> **난이도**: ⭐⭐ | **진단 초점**: L1, L2 | **전략**: pattern

### 풀이 1: 슬라이싱 활용 (pattern)

=== "코드"
    ```python
    text = input("문자열을 입력하세요: ")
    print(f"앞 3글자: {text[:3]}")
    print(f"뒤 3글자: {text[-3:]}")
    print(f"뒤집기: {text[::-1]}")
    ```

=== "핵심 아이디어"
    - `text[:3]`: 처음부터 3번째 직전까지 (인덱스 0, 1, 2).
    - `text[-3:]`: 뒤에서 3번째부터 끝까지.
    - `text[::-1]`: 전체를 역순으로 (step = -1).
    - 슬라이싱은 `[시작:끝:간격]` 형태입니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (슬라이싱) | 4줄 | ★★★★★ | ★★★★★ | 모든 학생 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 대괄호 닫지 않음 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    print(f"앞 3글자: {text[:3}")  # SyntaxError
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"앞 3글자: {text[:3]}")
    ```
    **원인**: f-string 중괄호 `{}`와 슬라이싱 대괄호 `[]`가 중첩되어 닫는 괄호를 누락합니다.

!!! danger "실수: 고정 인덱스로 뒤 3글자 접근 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    print(f"뒤 3글자: {text[4:7]}")  # 7글자일 때만 맞음!
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"뒤 3글자: {text[-3:]}")
    ```
    **원인**: 특정 길이의 문자열에서만 맞는 고정 인덱스를 사용합니다. 다른 길이의 문자열에서는 `IndexError` 또는 잘못된 결과가 나옵니다. 음수 인덱스를 사용하면 길이에 관계없이 동작합니다.

!!! danger "실수: 앞 3글자를 뒤 3글자에도 복붙 (아키타입: ai-dependent)"
    **❌ 잘못된 코드:**
    ```python
    print(f"앞 3글자: {text[:3]}")
    print(f"뒤 3글자: {text[:3]}")  # 복사 후 수정 안 함!
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"앞 3글자: {text[:3]}")
    print(f"뒤 3글자: {text[-3:]}")
    ```
    **원인**: 첫 줄을 복사하여 붙여넣은 후, 슬라이싱 부분을 수정하지 않았습니다.

!!! danger "실수: 뒤집기를 3글자만 적용 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    print(f"뒤집기: {text[-3:][::-1]}")  # 뒤 3글자만 뒤집음
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"뒤집기: {text[::-1]}")  # 전체 뒤집기
    ```
    **원인**: "뒤 3글자"와 "뒤집기"를 혼동하여, 뒤 3글자만 역순으로 출력합니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: "Python"을 입력하여 Pyt, hon, nohtyP이 나오는 것을 확인할 수 있는가?
    - **L1 (이해)**: `[:3]`과 `[-3:]`의 차이, `[::-1]`의 의미를 설명할 수 있는가?
    - **L2 (적용)**: "가운데 3글자 뽑기" 등 다른 슬라이싱 과제를 수행할 수 있는가?
    - **L3 (전략)**: 슬라이싱이 리스트에도 동일하게 적용된다는 것을 연결할 수 있는가?

!!! tip "슬라이싱 시각화"
    ```
    문자열: P  y  t  h  o  n
    인덱스: 0  1  2  3  4  5
    음수:  -6 -5 -4 -3 -2 -1

    [:3]  → P  y  t
    [-3:] →          h  o  n
    [::-1]→ n  o  h  t  y  P
    ```
    이 표를 함께 그리며 "칸 사이의 번호"로 슬라이싱을 설명하면 효과적입니다.
