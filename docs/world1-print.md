# 🖨️ World 1 — 출력의 세계 해설

> 프로그래밍의 첫걸음, `print()` 함수로 화면에 출력하는 법을 배웁니다.
> 학생들이 "내가 쓴 코드가 실행된다"는 경험을 처음 하는 월드입니다.

---

## 🧩 안녕, 세상아! (r01)

> `print()` 함수로 "Hello, World!"를 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L0, L1 | **전략**: pattern

### 풀이 1: 기본 출력 (pattern)

=== "코드"
    ```python
    print("Hello, World!")
    ```

=== "핵심 아이디어"
    - `print()` 함수는 괄호 안의 내용을 화면에 출력합니다.
    - 문자열은 반드시 따옴표(`"` 또는 `'`)로 감싸야 합니다.
    - 쉼표와 느낌표까지 정확히 포함해야 합니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (기본 출력) | 1줄 | ★★★★★ | - | 모든 학생 |

!!! tip "작은 따옴표 vs 큰 따옴표"
    `print('Hello, World!')` 도 동일하게 동작합니다.
    파이썬에서는 `'`와 `"`가 동등하지만, 일관성을 위해 하나를 선택하도록 안내하세요.

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 따옴표 누락 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    print(Hello, World!)
    ```
    **✅ 올바른 코드:**
    ```python
    print("Hello, World!")
    ```
    **원인**: 문자열을 따옴표로 감싸야 한다는 개념이 아직 형성되지 않았습니다. `Hello`를 변수명으로 인식하여 `NameError`가 발생합니다.

!!! danger "실수: 괄호 누락 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    print "Hello, World!"
    ```
    **✅ 올바른 코드:**
    ```python
    print("Hello, World!")
    ```
    **원인**: Python 2 문법이나 다른 언어 경험의 영향. Python 3에서 `print`는 함수이므로 반드시 괄호가 필요합니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 코드를 입력하고 실행 버튼을 눌러 결과를 확인할 수 있는가?
    - **L1 (이해)**: `print()` 함수의 역할, 따옴표의 의미를 설명할 수 있는가?
    - **L2 (적용)**: 다른 문자열로 바꿔서 출력할 수 있는가?
    - **L3 (전략)**: 이 단계에서는 해당하지 않음 (단일 접근법)

!!! tip "첫 문제 교수 전략"
    첫 문제는 **성공 경험**이 가장 중요합니다. 문법 설명보다는 "실행 → 결과 확인" 사이클을 빠르게 경험하도록 유도하세요.

## 🧩 자기소개 카드 (r02)

> 3줄에 걸쳐 이름, 나이, 취미를 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1, L2 | **전략**: brute_force, pattern

### 풀이 1: 여러 번 출력 (brute_force)

=== "코드"
    ```python
    print("이름: 홍길동")
    print("나이: 15살")
    print("취미: 코딩")
    ```

=== "핵심 아이디어"
    - `print()`를 여러 번 호출하면 각각 한 줄씩 출력됩니다.
    - 가장 직관적이고 이해하기 쉬운 방법입니다.
    - 초보 학생에게 권장하는 첫 번째 접근법입니다.

### 풀이 2: 삼중 따옴표 (pattern)

=== "코드"
    ```python
    print("""이름: 홍길동
    나이: 15살
    취미: 코딩""")
    ```

=== "비교"
    - 삼중 따옴표(`"""`)를 사용하면 여러 줄 문자열을 한 번에 출력할 수 있습니다.
    - `print()` 호출이 1회로 줄어들어 코드가 간결합니다.
    - 단, 들여쓰기에 주의해야 합니다 (불필요한 공백이 포함될 수 있음).

### 풀이 3: 줄바꿈 문자 (pattern)

=== "코드"
    ```python
    print("이름: 홍길동\n나이: 15살\n취미: 코딩")
    ```

=== "비교"
    - `\n`은 줄바꿈(개행) 문자입니다.
    - 한 줄의 코드로 여러 줄 출력이 가능합니다.
    - 가독성은 떨어지지만, 이스케이프 문자의 개념을 학습할 수 있습니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| brute_force (개별 print) | 3줄 | ★★★★★ | ★★★ | 초보 학생 |
| pattern (삼중 따옴표) | 3줄 | ★★★★ | ★★★★ | 문법 탐구형 학생 |
| pattern (줄바꿈 문자) | 1줄 | ★★★ | ★★ | 심화 탐구 |

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 줄바꿈 없이 한 줄 출력 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    print("이름: 홍길동 나이: 15살 취미: 코딩")
    ```
    **✅ 올바른 코드:**
    ```python
    print("이름: 홍길동")
    print("나이: 15살")
    print("취미: 코딩")
    ```
    **원인**: "3줄에 걸쳐 출력"이라는 요구사항을 놓칩니다. 출력 결과를 확인하지 않고 제출하는 습관이 원인입니다.

!!! danger "실수: 삼중 따옴표 들여쓰기 문제 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    print("""이름: 홍길동
        나이: 15살
        취미: 코딩""")
    ```
    **✅ 올바른 코드:**
    ```python
    print("""이름: 홍길동
    나이: 15살
    취미: 코딩""")
    ```
    **원인**: 코드 들여쓰기와 문자열 내용의 들여쓰기를 혼동합니다. 삼중 따옴표 안의 공백은 모두 출력에 포함됩니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 3줄 출력 결과를 화면에서 확인할 수 있는가?
    - **L1 (이해)**: `print()`가 한 줄씩 출력한다는 것을 이해하는가?
    - **L2 (적용)**: 삼중 따옴표나 `\n` 등 다른 방법으로도 같은 결과를 만들 수 있는가?
    - **L3 (전략)**: 상황에 따라 어떤 출력 방식이 적합한지 판단할 수 있는가?

## 🧩 나만의 명함 (r03)

> setup_code로 제공된 `name`, `age` 변수를 활용하여 f-string으로 명함을 출력하세요.
>
> **난이도**: ⭐ | **진단 초점**: L1, L2 | **전략**: pattern

### 풀이 1: f-string 활용 (pattern)

=== "코드"
    ```python
    # setup_code에서 name, age 변수가 제공됩니다
    print(f"이름: {name}")
    print(f"나이: {age}살")
    print(f"안녕하세요, {name}({age}살)입니다!")
    ```

=== "핵심 아이디어"
    - f-string은 `f"..."` 형태로, 중괄호 `{}` 안에 변수를 넣으면 값이 자동으로 삽입됩니다.
    - setup_code에서 제공하는 변수를 **그대로 사용**해야 합니다 (직접 정의하면 안 됨).
    - 변수를 활용한 동적 출력의 첫 경험입니다.

### 접근법 비교

| 전략 | 코드 길이 | 가독성 | 확장성 | 추천 대상 |
|------|----------|--------|--------|----------|
| pattern (f-string) | 3줄 | ★★★★★ | ★★★★★ | 모든 학생 |

!!! note "왜 f-string만 권장하는가?"
    `format()` 메서드나 `%` 포맷팅도 가능하지만, 중학생~고1 대상에서는 f-string이 가장 직관적입니다. pythink2에서는 f-string을 표준으로 사용합니다.

### ⚠️ 학생들이 자주 하는 실수

!!! danger "실수: 닫는 따옴표 누락 (아키타입: syntax-stuck)"
    **❌ 잘못된 코드:**
    ```python
    print(f"이름: {name})
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"이름: {name}")
    ```
    **원인**: f-string의 따옴표와 괄호가 많아 짝을 맞추기 어렵습니다. 에디터의 자동 완성 기능 활용을 안내하세요.

!!! danger "실수: 변수 직접 정의 시도 (아키타입: eager-beginner)"
    **❌ 잘못된 코드:**
    ```python
    name = "홍길동"
    age = 15
    print(f"이름: {name}")
    ```
    **✅ 올바른 코드:**
    ```python
    # name, age는 setup_code에서 이미 제공됩니다
    print(f"이름: {name}")
    ```
    **원인**: setup_code의 개념을 이해하지 못하고, 변수를 직접 정의하려 합니다. 이 경우 하드코딩으로 판정되어 코드 검사를 통과하지 못합니다.

!!! danger "실수: 하드코딩 (아키타입: ai-dependent)"
    **❌ 잘못된 코드:**
    ```python
    print("이름: 홍길동")
    print("나이: 15살")
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"이름: {name}")
    print(f"나이: {age}살")
    ```
    **원인**: 변수를 사용하지 않고 값을 직접 문자열에 넣습니다. AI 코치의 답변을 이해 없이 복사하는 패턴에서 자주 발생합니다.

!!! danger "실수: f 접두사 누락 (아키타입: validation-weak)"
    **❌ 잘못된 코드:**
    ```python
    print("이름: {name}")
    ```
    **✅ 올바른 코드:**
    ```python
    print(f"이름: {name}")
    ```
    **원인**: `f` 접두사 없이 중괄호를 쓰면 `{name}`이 그대로 문자열로 출력됩니다. 결과를 확인하지 않고 제출하는 습관이 원인입니다.

### 🎯 교사 코칭 포인트

!!! info "사고 수준 판단"
    - **L0 (실행)**: 코드를 실행하여 명함이 출력되는 것을 확인할 수 있는가?
    - **L1 (이해)**: f-string의 `f`와 `{}` 의 역할을 설명할 수 있는가?
    - **L2 (적용)**: 다른 변수(학교, 반 등)를 추가하여 명함을 확장할 수 있는가?
    - **L3 (전략)**: 이 단계에서는 해당하지 않음 (단일 접근법)

!!! tip "setup_code 교수 전략"
    이 문제에서 학생은 처음으로 **"이미 존재하는 변수를 활용하는"** 경험을 합니다.
    "컴퓨터가 이미 `name`이라는 상자에 값을 넣어뒀어요. 여러분은 그 상자를 꺼내 쓰기만 하면 돼요." 라는 비유가 효과적입니다.
