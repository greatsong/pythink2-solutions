    📦 World 2 — 변수 마을 해설¶

    변수에 값을 저장하고, 꺼내 쓰고, 바꾸는 법을 배웁니다. "값에 이름을 붙인다"는 프로그래밍의 핵심 개념을 경험하는 월드입니다.

    🧩 보물 상자에 담기 (r04)¶

    setup_code로 제공된 fruit, count 변수를 활용하여 f-string으로 출력하세요.

    난이도: ⭐ | 진단 초점: L1 | 전략: pattern

    풀이 1: f-string 활용 (pattern)¶

    # setup_code에서 fruit, count 변수가 제공됩니다
print(f"과일: {fruit}")
print(f"개수: {count}개")
print(f"{fruit}이(가) {count}개 있습니다.")
    • r03에서 배운 f-string을 다른 변수로 연습합니다.
    • 문자열 변수(fruit)와 숫자 변수(count)를 동시에 사용합니다.
    • f-string은 변수 타입에 관계없이 자동으로 문자열로 변환합니다.

    접근법 비교¶

    전략 코드 길이 가독성 확장성 추천 대상
    pattern (f-string) 3줄 ★★★★★ ★★★★★ 모든 학생

    ⚠️ 학생들이 자주 하는 실수¶

    실수: 변수명 오타 (아키타입: syntax-stuck)

    ❌ 잘못된 코드: 

    print(f"과일: {Fruit}")
    ✅ 올바른 코드: 
    print(f"과일: {fruit}")
    원인: 파이썬은 대소문자를 구분합니다. Fruit과 fruit은 다른 변수입니다.

    실수: 하드코딩 (아키타입: strategy-biased)

    ❌ 잘못된 코드: 

    print("과일: 사과")
print("개수: 5개")
    ✅ 올바른 코드: 
    print(f"과일: {fruit}")
print(f"개수: {count}개")
    원인: setup_code 변수를 활용하지 않고, 이전 문제(r02)처럼 직접 값을 넣는 방식을 고수합니다.

    🎯 교사 코칭 포인트¶

    사고 수준 판단

    • L0 (실행): 코드 실행 결과를 확인할 수 있는가?
    • L1 (이해): fruit과 count가 각각 어떤 값을 담고 있는지 설명할 수 있는가?
    • L2 (적용): 새로운 변수를 추가하여 출력을 확장할 수 있는가?
    • L3 (전략): 이 단계에서는 해당하지 않음

    🧩 마법의 컵 바꾸기 (r05)¶

    두 변수 a와 b의 값을 서로 바꿔서 출력하세요.

    난이도: ⭐⭐ | 진단 초점: L2, L3 | 전략: pattern, brute_force

    풀이 1: 파이썬 튜플 스왑 (pattern)¶

    # setup_code에서 a, b 변수가 제공됩니다
a, b = b, a
print(f"a = {a}")
print(f"b = {b}")
    • 파이썬만의 독특한 문법: a, b = b, a로 한 줄에 두 변수를 교환합니다.
    • 내부적으로 튜플 패킹/언패킹이 일어납니다.
    • 임시 변수가 필요 없어 간결합니다.

    풀이 2: 임시 변수 활용 (brute_force)¶

    # setup_code에서 a, b 변수가 제공됩니다
temp = a
a = b
b = temp
print(f"a = {a}")
print(f"b = {b}")
    • 대부분의 프로그래밍 언어에서 사용하는 전통적인 스왑 방법입니다.
    • 임시 변수(temp)에 원래 값을 백업한 뒤 교환합니다.
    • 코드가 길지만, "왜 임시 변수가 필요한지" 사고 과정이 명확합니다.
    • 알고리즘적 사고의 기초가 되는 패턴입니다.

    접근법 비교¶

    전략 코드 길이 가독성 확장성 추천 대상
    pattern (튜플 스왑) 3줄 ★★★★★ ★★★ 파이썬 관용구에 익숙한 학생
    brute_force (임시 변수) 5줄 ★★★★ ★★★★★ 논리적 사고를 연습하는 학생

    ⚠️ 학생들이 자주 하는 실수¶

    실수: 콜론 오용 (아키타입: syntax-stuck)

    ❌ 잘못된 코드: 

    a, b = b, a:
print(f"a = {a}")
    ✅ 올바른 코드: 
    a, b = b, a
print(f"a = {a}")
    원인: 콜론(:)을 if문이나 for문처럼 모든 줄 끝에 붙여야 한다고 오해합니다.

    실수: 할당 순서 오류 (아키타입: eager-beginner)

    ❌ 잘못된 코드: 

    a = b
b = a
print(f"a = {a}")
print(f"b = {b}")
    ✅ 올바른 코드: 
    temp = a
a = b
b = temp
print(f"a = {a}")
print(f"b = {b}")
    원인: a = b를 실행하면 a의 원래 값이 사라집니다. 그 후 b = a를 하면 두 변수 모두 같은 값이 됩니다. 이것이 임시 변수가 필요한 이유입니다.

    실수: 출력 포맷 오류 (아키타입: validation-weak)

    ❌ 잘못된 코드: 

    a, b = b, a
print(a, b)
    ✅ 올바른 코드: 
    a, b = b, a
print(f"a = {a}")
print(f"b = {b}")
    원인: 교환은 성공했지만, 요구하는 출력 형식을 맞추지 않았습니다.

    🎯 교사 코칭 포인트¶

    사고 수준 판단

    • L0 (실행): 코드를 실행하여 값이 바뀐 것을 확인할 수 있는가?
    • L1 (이해): 왜 a = b; b = a가 작동하지 않는지 설명할 수 있는가?
    • L2 (적용): 임시 변수 방식으로 3개 변수를 순환 교환할 수 있는가?
    • L3 (전략): 튜플 스왑과 임시 변수 방식의 차이를 비교하고, 각각의 장점을 설명할 수 있는가?

    토론 유도 질문

    "만약 파이썬이 아닌 다른 언어를 쓴다면, a, b = b, a를 쓸 수 있을까요?" 이 질문으로 언어별 특성과 알고리즘의 보편성에 대한 토론을 유도할 수 있습니다.

    🧩 용돈 계산기 (r06)¶

    setup_code로 제공된 money(용돈)와 rate(저축률%)를 활용하여 저축액과 남은 용돈을 계산하세요.

    난이도: ⭐ | 진단 초점: L1, L2 | 전략: math, decomposition

    풀이 1: 수학 공식 직접 적용 (math)¶

    # setup_code에서 money, rate 변수가 제공됩니다
savings = int(money * rate / 100)
remaining = money - savings
print(f"용돈: {money}원")
print(f"저축률: {rate}%")
print(f"저축액: {savings}원")
print(f"남은 용돈: {remaining}원")
    • 백분율 계산: money * rate / 100
    • int()로 소수점을 버려 정수 원 단위로 만듭니다.
    • 남은 돈 = 전체 - 저축액 으로 간단하게 계산합니다.

    풀이 2: 단계별 분해 (decomposition)¶

    # setup_code에서 money, rate 변수가 제공됩니다
# 1단계: 비율을 소수로 변환
ratio = rate / 100

# 2단계: 저축액 계산 (정수로 변환)
savings = int(money * ratio)

# 3단계: 남은 금액 계산
remaining = money - savings

# 4단계: 결과 출력
print(f"용돈: {money}원")
print(f"저축률: {rate}%")
print(f"저축액: {savings}원")
print(f"남은 용돈: {remaining}원")
    • 계산 과정을 단계별로 분리하여 각 단계의 의미를 명확히 합니다.
    • ratio 변수를 추가하여 "비율 변환"과 "곱셈"을 분리합니다.
    • 코드는 길어지지만, 디버깅과 이해가 쉽습니다.

    접근법 비교¶

    전략 코드 길이 가독성 확장성 추천 대상
    math (직접 공식) 6줄 ★★★★ ★★★ 수학에 자신 있는 학생
    decomposition (단계 분해) 10줄 ★★★★★ ★★★★★ 논리적 분해를 연습하는 학생

    ⚠️ 학생들이 자주 하는 실수¶

    실수: int() 미사용으로 소수점 출력 (아키타입: validation-weak)

    ❌ 잘못된 코드: 

    savings = money * rate / 100
print(f"저축액: {savings}원")  # 출력: 저축액: 15000.0원
    ✅ 올바른 코드: 
    savings = int(money * rate / 100)
print(f"저축액: {savings}원")  # 출력: 저축액: 15000원
    원인: 나눗셈(/)의 결과는 항상 float입니다. 원 단위는 정수여야 자연스러우므로 int()로 변환해야 합니다.

    실수: 계산 순서 오류 (아키타입: eager-beginner)

    ❌ 잘못된 코드: 

    savings = int(money / 100 * rate)  # 순서는 같지만...
remaining = money - rate  # rate를 빼버림!
    ✅ 올바른 코드: 
    savings = int(money * rate / 100)
remaining = money - savings
    원인: remaining에서 저축액(savings) 대신 저축률(rate)을 빼는 실수. 변수의 의미를 정확히 파악하지 못한 경우입니다.

    🎯 교사 코칭 포인트¶

    사고 수준 판단

    • L0 (실행): 코드 실행 결과를 확인하고, 계산이 맞는지 손으로 검산할 수 있는가?
    • L1 (이해): int()와 /의 관계, 백분율 계산의 원리를 설명할 수 있는가?
    • L2 (적용): 세금 계산, 할인가 계산 등 유사한 문제에 같은 패턴을 적용할 수 있는가?
    • L3 (전략): 한 줄 계산과 단계별 분해의 장단점을 비교할 수 있는가?
